# MetaNutri 前端项目优化方案

## 一、代码质量与架构优化

### 1. TypeScript 类型定义完善
- **问题**：项目使用 `.js` 文件，缺少类型定义，容易出现运行时错误
- **方案**：将核心文件逐步迁移为 `.tsx`，为 API 响应、组件 props 添加接口定义
- **优先级**：P0

### 2. 状态管理重构
- **问题**：状态管理分散在各个组件中，数据共享困难
- **方案**：引入 Zustand 或 React Context 集中管理用户状态、语言状态、全局 UI 状态
- **优先级**：P1

### 3. API 请求统一封装
- **问题**：`lib/api.js` 缺少统一的错误处理和请求拦截器
- **方案**：
  - 创建 `axios` 实例，配置请求/响应拦截器
  - 添加 Token 自动刷新机制
  - 统一错误处理和日志记录
- **优先级**：P0

### 4. 自定义 Hooks 封装
- **问题**：重复逻辑未抽取为 hooks
- **方案**：
  - `useAuth`：处理登录状态、Token、用户信息
  - `useFetch`：封装数据请求逻辑
  - `useLanguage`：已存在，需扩展到全局
- **优先级**：P2

---

## 二、性能优化

### 1. 组件懒加载优化
- **问题**：仅 Dashboard 页面使用了动态导入，其他页面未优化
- **方案**：对大数据可视化组件（ECharts）、BioBackground 等进行懒加载，添加加载骨架屏
- **优先级**：P1

### 2. 图片资源优化
- **问题**：缺少图片优化策略
- **方案**：使用 Next.js 内置的 `next/image` 组件，配置图片质量、格式转换（WebP/AVIF）
- **优先级**：P2

### 3. BioBackground 动画性能优化
- **问题**：多个 SVG 动画同时运行可能影响性能
- **方案**：
  - 使用 `will-change` 属性优化动画渲染
  - 通过 `requestAnimationFrame` 统一调度
  - 根据设备性能动态调整动画数量
- **优先级**：P2

### 4. 缓存策略增强
- **问题**：API 响应未缓存，重复请求浪费资源
- **方案**：使用 `SWR` 或 `React Query` 实现数据缓存、自动重新验证、乐观更新
- **优先级**：P1

## 三、用户体验优化

### 1. 全局 Loading 状态
- **问题**：缺少统一的加载状态反馈
- **方案**：创建全局 Loading 组件，在页面切换和数据请求时显示
- **优先级**：P0

### 2. 错误边界（Error Boundary）
- **问题**：组件错误会导致整个页面崩溃
- **方案**：实现 React Error Boundary，捕获错误并显示友好的错误页面
- **优先级**：P0

### 3. Toast 通知系统
- **问题**：操作反馈缺失，用户不知道操作是否成功
- **方案**：集成 `react-hot-toast` 或自定义 Toast 组件，提供成功/错误/警告通知
- **优先级**：P0

### 4. 表单验证增强
- **问题**：登录/注册表单缺少前端验证
- **方案**：使用 `react-hook-form` + `zod` 实现表单验证，实时显示错误提示
- **优先级**：P1

---

## 四、国际化（i18n）完善

### 1. 全页面国际化
- **问题**：仅首页实现了国际化，其他页面未适配
- **方案**：将 i18n 扩展到所有页面和组件
- **优先级**：P0

### 2. 语言持久化
- **问题**：语言设置未持久化，刷新页面后重置
- **方案**：将语言设置保存到 `localStorage`，初始化时读取
- **优先级**：P0

### 3. RTL 支持
- **问题**：未考虑阿拉伯语等从右到左语言
- **方案**：添加 RTL 样式支持，使用 `dir` 属性切换文本方向
- **优先级**：P3

---

## 五、安全优化

### 1. Token 安全管理
- **问题**：Token 直接存储在 `localStorage`，易受 XSS 攻击
- **方案**：考虑使用 `httpOnly` Cookie 或更安全的存储方案
- **优先级**：P1

### 2. 请求安全头
- **问题**：缺少安全响应头配置
- **方案**：在 `next.config.js` 中配置 `headers`，添加 CSP、X-Frame-Options 等安全头
- **优先级**：P2

### 3. 输入安全
- **问题**：缺少 XSS 防护
- **方案**：对用户输入进行转义处理，使用 `dangerouslySetInnerHTML` 时谨慎处理
- **优先级**：P2

---

## 六、开发体验优化

### 1. ESLint 配置完善
- **问题**：项目有 lint 脚本但缺少自定义配置
- **方案**：创建 `.eslintrc.json`，配置 React 规则、import 排序、hooks 规则
- **优先级**：P2

### 2. Prettier 配置
- **问题**：缺少代码格式化工具配置
- **方案**：创建 `.prettierrc`，统一代码风格
- **优先级**：P2

### 3. Git Hooks
- **问题**：缺少提交前的代码检查
- **方案**：使用 `husky` + `lint-staged`，在提交前自动运行 lint 和 format
- **优先级**：P2

---

## 七、功能完整性优化

### 1. 用户头像上传
- **问题**：Profile 页面缺少头像功能
- **方案**：添加头像上传组件，支持图片预览和裁剪
- **优先级**：P2

### 2. 密码找回功能
- **问题**：登录页面缺少密码找回入口
- **方案**：添加忘记密码流程，通过邮箱发送重置链接
- **优先级**：P2

### 3. 数据可视化增强
- **问题**：Dashboard 图表较为基础
- **方案**：
  - 添加交互式图表（可缩放、筛选）
  - 优化图表配色和动画效果
  - 添加数据导出功能（CSV/PDF）
- **优先级**：P2

### 4. 移动端适配优化
- **问题**：部分页面移动端显示不够友好
- **方案**：优化移动端布局，确保关键操作触手可及
- **优先级**：P2

---

## 八、SEO 优化

### 1. 页面 Meta 标签
- **问题**：各页面缺少独立的 `metadata` 配置
- **方案**：为每个页面配置独立的 `title`、`description`、`keywords`
- **优先级**：P2

### 2. Open Graph 标签
- **问题**：缺少社交分享支持
- **方案**：添加 OG 标签，优化社交媒体分享效果
- **优先级**：P3

---

## 优化优先级总结

| 优先级 | 优化项 |
|--------|--------|
| **P0** | TypeScript 类型定义、API 请求封装、Toast 通知、错误边界、全页面国际化、语言持久化、全局 Loading |
| **P1** | 状态管理重构、缓存策略、组件懒加载、表单验证、Token 安全管理 |
| **P2** | 自定义 Hooks、图片优化、动画性能、ESLint/Prettier、Git Hooks、用户头像、密码找回、数据可视化增强、移动端适配、SEO Meta 标签 |
| **P3** | RTL 支持、安全响应头、输入安全、Open Graph 标签 |
